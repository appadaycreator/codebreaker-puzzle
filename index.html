シーザー暗号とは？

アルファベットを一定の数だけずらす暗号です。

HELLO → KHOOR（3文字ずらし）

A→D, B→E, C→F … のようにずらします

  2. 「暗号タイプを見る」で種類を確認する

暗号の種類 Cipher Types

🔐

シーザー暗号（Lv.1・2） Caesar Cipher (Lv.1&2)

アルファベットを一定数ずらす最古の暗号のひとつ。Julius Caesarが古代ローマで使用したことが名前の由来。HELLO → KHOOR（3文字ずらし）のように変換する。 One of the oldest ciphers — shifts letters by a fixed number. Named after Julius Caesar who used it in ancient Rome. HELLO → KHOOR (shift by 3).

🔄

換字式暗号（Lv.3） Substitution Cipher (Lv.3)

各文字を別の特定の文字に置き換える暗号。鍵となる対応表が必要で、S→Q, E→R のように決まったルールで変換する。シーザー暗号より解読が難しい。 Replaces each letter with another specific letter using a key table (S→Q, E→R). Harder to crack than Caesar since the shift varies per letter.

📡

モールス信号（Lv.4） Morse Code (Lv.4)

点（.）と線（-）の組み合わせで文字を表す通信方式。19世紀に発明され、船舶・航空・軍事通信で長く使われた。Eは「.」、Tは「-」と最短の符号を持つ。 Encodes letters as dots (.) and dashes (-). Invented in the 19th century and widely used in maritime, aviation, and military communications.

↩️

逆順暗号（Lv.5） Reverse Cipher (Lv.5)

文字列を後ろから前に並べ直すだけのシンプルな暗号。CRYPTOLOGY → YGOLOTPYRC のように変換する。最もわかりやすい暗号のひとつ。 Simply reverses the character order. CRYPTOLOGY → YGOLOTPYRC. One of the easiest ciphers to recognize and decode.

🔀

アナグラム（Lv.6） Anagram (Lv.6)

単語内の文字の順序を入れ替えた暗号。OPEN SESAME → PEN OSESAEM のように同じ文字を使って別の並び順にする。パズル的な楽しさがある。 Rearranges the letters within words. OPEN SESAME → PEN OSESAEM. Uses the same characters in a different order — fun puzzle-like quality.

🗝️

ヴィジュネル暗号（Lv.7） Vigenère Cipher (Lv.7)

キーワードを使ってシーザー暗号を組み合わせた多表式暗号。16世紀に考案され「解読不能の暗号」と呼ばれた。キー「KEY」を繰り返し使い各文字のずらし量を変える。 A polyalphabetic cipher using a keyword. Devised in the 16th century and called "le chiffre indéchiffrable." The keyword (e.g., KEY) cycles and shifts each letter differently.

🔢

数字変換（Lv.8） Number Conversion (Lv.8)

A=1, B=2, … Z=26 として文字を数字に変換する暗号。「5 14 9 7 13 1」→「ENIGMA」のように読む。単純だが見た目がまったく変わる。 Converts letters to numbers: A=1, B=2, … Z=26. "5 14 9 7 13 1" decodes to "ENIGMA." Simple rule but dramatically changes the appearance.

💻

Base64（Lv.9） Base64 (Lv.9)

バイナリデータをASCII文字64種類で表現するエンコード方式。メール添付・Web API・画像のインライン埋め込みなど現代のWebで広く使われている。 Encodes binary data using 64 ASCII characters. Widely used in modern web for email attachments, API data, and inline image embedding.

♻️

ROT13（Lv.10） ROT13 (Lv.10)

アルファベットを13文字ずらすシーザー暗号の特殊版。最大の特徴は「同じ操作で暗号化も復号もできる」こと。THANK YOU → GUNAX LBH。インターネット掲示板でネタバレ隠しに使われた。 A special Caesar cipher that shifts by 13. Its key property: applying ROT13 twice returns the original text. THANK YOU → GUNAX LBH. Used on forums to hide spoilers.

🪞

アトバシュ暗号（Lv.11） Atbash Cipher (Lv.11)

アルファベットを鏡のように逆にする暗号（A↔Z, B↔Y...）。古代ヘブライ語の暗号として聖書にも登場する。GREAT JOB → TIVZG QLY のように変換する。 A cipher that mirrors the alphabet (A↔Z, B↔Y...). Originates from ancient Hebrew and appears in the Bible. GREAT JOB → TIVZG QLY.

🖥️

バイナリ変換（Lv.12） Binary Code (Lv.12)

コンピュータが扱う2進数で文字を表現する方式。A=01000001（10進数65）のように各文字は8ビットで表される。「01010111 01001001 01001110」→「WIN」と読む。 Represents each character as 8-bit binary (A=01000001). "01010111 01001001 01001110" → "WIN". The foundation of all digital data storage and transmission.

📻

フォネティックコード（Lv.13） Phonetic Code (Lv.13)

無線通信でアルファベットを音声で確実に伝えるためのNATO標準コード。Alpha=A, Bravo=B... の頭文字が文字を表す。「ALPHA LIMA INDIA CHARLIE ECHO」→「ALICE」。 NATO phonetic alphabet used to clearly transmit letters over radio. Alpha=A, Bravo=B... "ALPHA LIMA INDIA CHARLIE ECHO" → "ALICE".

🔁

ROT7暗号（Lv.14） ROT7 Cipher (Lv.14)

アルファベットを7文字ずらすシーザー暗号の変形。Z→Gのように末尾を超えると先頭に折り返す。「CODE MASTER」→「JVKL THZALY」。ROT13より解読しにくい。 A Caesar cipher shifted by 7. Wraps around when reaching Z (e.g., T+7=A). "CODE MASTER" → "JVKL THZALY". Slightly harder than ROT13 to crack without tools.

🔢

逆順数字暗号（Lv.15） Reverse Number Cipher (Lv.15)

Z=1, Y=2... A=26 と通常のA=1と逆順で数字を割り当てる暗号。「4 22 15 15 23 12 13 22」→「WELL DONE」。Lv.8の数字変換とは逆の対応表を使う。 Assigns numbers in reverse order: Z=1, Y=2... A=26. "4 22 15 15 23 12 13 22" → "WELL DONE". The reverse of the number cipher in Lv.8.

暗号攻略ガイド Cipher Strategy Guide

🟢

初級（Lv.1〜5）の攻略法 Easy (Lv.1–5) Strategy

  • シーザー暗号はA〜Zを1〜3文字ずらして試す
  • Try shifting A–Z by 1–3 letters for Caesar cipher
  • 逆順暗号は文字列を後ろから読むだけ
  • Reverse cipher: just read the string backwards
  • 短い単語（3〜4文字）から推測する
  • Start guessing from short words (3–4 letters)
🟡

中級（Lv.6〜10）の攻略法 Medium (Lv.6–10) Strategy

  • モールス信号は「/」で単語が区切れる
  • Morse: "/" separates words
  • 数字変換はA=1からZ=26の表に沿って変換
  • Number cipher: A=1 through Z=26
  • ROT13はブラウザコンソールで即確認できる
  • ROT13 can be tested instantly in browser console
🔴

上級（Lv.11〜15）の攻略法 Hard (Lv.11–15) Strategy

  • バイナリは8桁ずつ10進数に変換してASCIIコード表と照合
  • Binary: convert each 8-bit group to decimal, then to ASCII
  • ヴィジュネルはKEYの各文字のアルファベット順番号分逆にずらす
  • Vigenère: shift each letter backward by the key letter's position
  • フォネティックコードは各単語の先頭文字だけ読む
  • Phonetic: take only the first letter of each word
